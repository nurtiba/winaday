## Vision

### 10x Check

WinTheDay at 10x is not a tracker. It's a daily 30-second mirror that makes your life legible to yourself and others. The GitHub contribution graph proved that visualization creates accountability at scale. WinTheDay applies that same mechanic to the five dimensions of a good life. The 10x version: you open your phone, score your day in 30 seconds, see your year light up, and share a beautiful card that makes your friends want to do the same. Strava for your whole life.

The current MVP has the core loop: score, see heatmap, share card. But it's missing the retention mechanics (streaks with teeth), the onboarding that enables everything (Add-to-Home-Screen), the safety net for validation (data export), and the polish that makes sharing actually happen (share card design).

## Accepted Scope (added to this plan)

### 1. Add-to-Home-Screen Onboarding

Detect if running in browser vs standalone PWA via `window.matchMedia('(display-mode: standalone)')`. Show a guided prompt after first scoring session.

**Two code paths required:**
- **Android/Chrome:** Use the deferred `beforeinstallprompt` event. Listen for the event on page load, stash it, trigger `.prompt()` after first score. Note: `beforeinstallprompt` is one-shot per session. If user dismisses the native browser prompt and later taps the footer link, show the static instructional overlay (same as iOS) instead of calling `.prompt()` again. If the event never fires (already installed, or unsupported browser), silently suppress the prompt.
- **iOS Safari:** No install API exists. Show a static instructional overlay with step-by-step: "Tap the Share button (icon) > Scroll down > Tap 'Add to Home Screen'." Use device detection (`navigator.userAgent` for Safari on iOS) to show the correct flow.

**Dismiss behavior:** Store `localStorage` key `wintheday_a2hs_dismissed = true` on dismiss. Never re-show automatically (user must manually re-enable). Show "Add to Home Screen" link on calendar page footer for users who dismissed. Tapping this link re-shows the instructional overlay.

**Acceptance criteria:** Prompt appears after first score submission on mobile Safari/Chrome. Absent in standalone mode. Dismiss persists across sessions. Footer link works even if browser prompt was previously dismissed.

### 2. Streak Gamification

Milestone celebrations at 3, 7, 14, 30, 60, 100, 365 days. Hard streak reset on miss (no grace period for MVP validation, simplifies logic). "Streak at risk" warning shown on the home page after 9 PM local time if user has not scored today.

**Day boundary:** Use `getToday()` from `storage.ts` exclusively for all streak logic. Do not call `toISOString()` directly in streak code. `getToday()` returns the local-timezone date string. A streak breaks when there is no score for the previous calendar day at the time of the next score.

**"At risk" check:** Uses `new Date().getHours() >= 21` (local time, `getHours()` is already local). Warning dismisses immediately upon score submission for the current day.

**Celebration animation:** Full-screen confetti burst (CSS keyframe animation, no library), 2 seconds duration, auto-dismisses. Confetti layer has `pointer-events: none` so taps pass through to elements underneath. Shows milestone number ("7 days!") centered.

**Acceptance criteria:** Streak counter increments on consecutive days. Resets on gap > 1 day. Confetti shows on milestone days. "At risk" warning visible after 9 PM if unscored, hidden once scored.

### 3. Data Export/Import

"Download my data" button on the calendar page. Exports all `DayScore` objects as a JSON file (`wintheday-backup-YYYY-MM-DD.json`).

"Restore from backup" button next to it. Opens file picker, reads JSON, **overwrites** current data with backup (with confirmation dialog: "This will replace all your current scores. Continue?"). No merge logic for MVP... keeps it simple and safe.

**Error handling:** If JSON parse fails or the array is empty/malformed, show inline error: "Invalid backup file. No data was imported." Do not overwrite existing data on error.

**Acceptance criteria:** Export downloads a valid JSON file. Import replaces all data after confirmation. Malformed files show error without data loss. Imported data renders correctly in heatmap and score views.

### 4. Share Card Design Polish

Run `/design-review` on the live share card after implementation. Concrete deliverables:
- **Contrast:** All text must meet WCAG AA (4.5:1 contrast ratio against card background)
- **Spacing:** Tighten padding between category bars, ensure score percentage is the visual focal point
- **Branding strip:** "WinTheDay" text at bottom, clean, no logo image dependency (text-only for html2canvas reliability)
- **html2canvas reliability:** Test on iOS Safari. Known issues: `backdrop-filter`, `box-shadow` with spread, cross-origin images, and CSS `filter` all fail silently. Avoid these CSS properties in the share card. **Blank detection:** After `html2canvas` resolves, check if `canvas.width === 0 || canvas.height === 0`, then sample center pixel alpha. If blank, fall back to showing a "Screenshot this screen" instruction instead of a download.

**Acceptance criteria:** Share card renders identically on Chrome and iOS Safari. All text passes WCAG AA contrast. html2canvas produces a downloadable image. Fallback instruction shown if capture fails.

## Bug Fixes (from outside voice review)

### Fix 1: Timezone bug in formatDate()
`storage.ts:78-80` uses `toISOString().split('T')[0]` which returns UTC date, not local. Replace with local-timezone formatting: `const year = date.getFullYear(); const month = String(date.getMonth() + 1).padStart(2, '0'); const day = String(date.getDate()).padStart(2, '0'); return \`${year}-${month}-${day}\`;`. This fixes streaks, milestones, heatmap dates, and "at risk" warnings for all timezones.

### Fix 2: ShareCard gradient text invisible in html2canvas
`ShareCard.tsx:65,75` uses `WebkitTextFillColor: "transparent"` with gradient clip. html2canvas renders this as invisible text on many devices. Replace with solid color (`#4ade80` green) for the score number and percent sign in ShareCard only. Keep the gradient in `result/page.tsx` (screen-only, not captured by html2canvas).

### Fix 3: beforeinstallprompt global listener
The `beforeinstallprompt` event fires once, early in page load. Must be caught in `layout.tsx` or a global context/hook, not inside a page component. Stash the event in a React context (`A2HSContext`) so any component can trigger `.prompt()`.
